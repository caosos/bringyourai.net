# Canonical Project Registry

**Date created:** 2026-05-17  
**Owner:** Michael Chambers  
**Repository:** caosos/bringyourai.net  
**Registry status:** Canonical working registry v1  
**Scope:** Consolidates recurring CAOS / BringYourAI / CAOS Care project ideas into a structured, timestamped project map.

## Purpose

This folder exists because the same underlying architecture has been appearing across multiple conversations, emails, proposals, and outreach threads under different names.

The common architecture is:

> Persistent contextual intelligence layered onto existing human systems, devices, buildings, workflows, institutions, and machines.

The purpose of this registry is to stop idea drift by giving each project family a named home, priority level, timestamp, and decision status.

## Priority definitions

- **P0:** Foundational. Must exist for other projects to work.
- **P1:** Highest practical/commercial priority.
- **P2:** Strong project, depends on P0/P1 foundation.
- **P3:** Strategic longer-range expansion.
- **P4:** Research/parking-lot concept until evidence or partner emerges.

## Current top priority

The highest-priority commercialization path is:

1. CAOS Core Platform foundation.
2. CAOS Care as first vertical.
3. Retrofit building / hardware partner layer.
4. BringYourAI command layer as broader consumer/business packaging.

This order preserves the big vision while anchoring it in a reachable first market.
